Taro Study

发表于2021-01-03|更新于2021-07-28
|阅读量:

一、是什么？

taro的概念：

Taro是一个开放式跨端跨框架解决方案。

二、有什么用？（或者说为什么要使用）

优点（特点）：

1. 一套代码适配到多端（最大的优点）

2. 文档较为健全明了，无论是升级迁移或者学习使用，框架可维护性较强

3. 生态相比其他框架较为完善

支持使用 React/Vue/Nerv 等框架来开发 微信 / 京东 / 百度 / 支付宝 / 字节跳动 / QQ 小程序 / H5 等应用。

三、如何使用？（这里介绍3.X的版本）

1. 安装及使用

官方链接：https://taro-docs.jd.com/taro/docs/GETTING-STARTED

注意：Taro 项目基于 node，请确保已具备较新的 node 环境（>=12.0.0）

运行

常用taro cli命令

微信小程序开发开发前注意事项

若使用 微信小程序预览模式 ，则需下载并使用微信开发者工具添加项目进行预览，此时需要注意微信开发者工具的项目设置

  • 需要设置关闭 ES6 转 ES5 功能，开启可能报错
  • 需要设置关闭上传代码时样式自动补全，开启可能报错
  • 需要设置关闭代码压缩上传，开启可能报错

2. 基本使用

官方基础教程：https://taro-docs.jd.com/taro/docs/react

app.config.js 的默认导出就是全局配置

3.路由

在 Taro 中，路由功能是默认自带的，不需要开发者进行额外的路由配置。

我们只需要在入口文件的 config 配置中指定好 pages，然后就可以在代码中通过 Taro 提供的 API 来跳转到目的页面

路由详情请见官方文档：https://nervjs.github.io/taro/docs/apis/route/navigateTo

4. 编译配置

编译配置存放于项目根目录下 config 目录中，包含三个文件

  • index.js 是通用配置
  • dev.js 是项目预览时的配置
  • prod.js 是项目打包时的配置

编译配置详情见：https://taro-docs.jd.com/taro/docs/config-detail

5. 其他

hooks在taro中的使用： https://taro-docs.jd.com/taro/docs/hooks

预渲染：https://taro-docs.jd.com/taro/docs/prerender

长列表渲染：https://taro-docs.jd.com/taro/docs/virtual-list

redux在taro中的使用： https://taro-docs.jd.com/taro/docs/redux

CSS Modules在taro中的使用：https://taro-docs.jd.com/taro/docs/css-modules

注意：如果在 Taro 项目引用了小程序原生的第三方组件和插件，那么该项目将不再具备多端转换的能力，例如，如果使用了微信小程序的第三方组件，那么项目只能转换成微信小程序，转义成其他平台会失效，使用其他小程序原生组件同理。
